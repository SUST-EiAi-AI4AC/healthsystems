# 健康报告历史记录功能使用说明

## 📋 功能概述

健康报告历史记录功能允许用户查看过往的健康报告，支持按时间范围筛选，并提供详细的历史报告查看功能。

## ✨ 主要功能

### 1. 历史记录入口
- 在健康报告页面顶部"综合健康报告"文字右侧添加了📅历史记录按钮
- 点击按钮即可打开历史记录弹窗

### 2. 时间筛选功能
提供四种时间筛选选项：
- **全部**：显示所有历史报告（最近 90 天）
- **按日**：显示最近 7 天的每日报告
- **按周**：显示最近 4 周的周报
- **按月**：显示最近 3 个月的月报

### 3. 历史报告列表
每条历史记录显示：
- 报告日期和星期几
- 总体健康评分（带颜色标识）
- 健康等级（优秀/良好/一般/需关注）
- 关键指标摘要（心率、睡眠、步数、SDS 评分）
- 数据来源标签

### 4. 历史报告详情
点击任意历史记录可查看完整详情，包括：
- **总体健康评分**：圆形可视化展示，配等级和建议
- **心理量表评估**：SDS（抑郁）、PSS（压力）、SAS（焦虑）的得分和等级
- **智能手环数据**：心率、睡眠、步数、血氧、压力、身体能量等 6 项指标
- **个性化健康建议**：根据当时数据生成的专属建议

### 5. 导出分享功能
- 点击"导出报告"可将历史报告以文本格式分享
- 支持微信分享或复制到剪贴板

## 🎨 UI 设计特点

### 配色方案
- 优秀（85+）：绿色渐变 `#43e97b → #38f9d7`
- 良好（70-84）：蓝色渐变 `#4facfe → #00f2fe`
- 一般（55-69）：橙色渐变 `#fda085 → #f6d365`
- 需关注（<55）：粉色渐变 `#f093fb → #f5576c`

### 交互体验
- 弹窗从底部滑入动画
- 按钮点击有缩放反馈
- 详情弹窗淡入效果
- 卡片式列表设计，易于浏览

## 💾 数据存储

### 存储位置
- 使用 `localStorage` 存储
- Key: `health_report_history`

### 数据保留策略
- 自动保存每次生成的健康报告
- 同一天只保存一份报告（避免重复）
- 只保留最近 90 天的记录
- 自动清理过期数据

## 📱 使用流程

### 查看历史记录
1. 打开健康报告页面（`/pages/mine/HealthReport.vue`）
2. 点击右上角的📅"历史记录"按钮
3. 浏览历史报告列表

### 筛选历史报告
1. 在历史记录弹窗顶部选择时间范围
2. 列表会自动刷新显示符合条件的报告
3. 可切换不同筛选条件查看不同时期的数据

### 查看历史详情
1. 在历史列表中点击任意报告卡片
2. 查看详情弹窗，包含完整的健康数据
3. 上下滑动查看所有内容

### 导出历史报告
1. 在详情页面底部点击"导出报告"按钮
2. 选择分享方式（微信或复制）
3. 分享给他人或保存到笔记

## 🎯 技术实现亮点

### 1. 自动保存机制
- 每次加载健康报告时自动保存到历史记录
- 智能去重：同一天只保存一次
- 定期清理：自动删除 90 天前的旧数据

### 2. 性能优化
- 使用虚拟滚动（scroll-view）处理长列表
- 按需加载：只在需要时渲染详情内容
- 数据缓存：利用 localStorage 减少网络请求

### 3. 用户体验
- 直观的视觉反馈（颜色、动画）
- 清晰的信息层级（标题、副标题、标签）
- 便捷的操作（一键导出、快速筛选）

### 4. 数据可视化
- 评分渐变色直观展示健康状态
- 图标 + 数字的组合易于理解
- 网格布局充分利用屏幕空间

## ⚠️ 注意事项

1. **首次使用**：如果没有历史数据，会显示空状态提示
2. **数据同步**：当前数据存储在本地，更换设备不会同步
3. **存储空间**：每条记录约 1-2KB，90 天记录约占用 50-100KB
4. **隐私保护**：敏感健康数据仅存储在本地，不会上传服务器
