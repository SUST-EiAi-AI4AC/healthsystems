# 新增功能测试指南

## 前置准备

### 1. 数据库检查
运行 `新增功能数据库检查.sql` 脚本，确保所有必需的表都存在：
- `ai_chat_session` - AI聊天会话表
- `ai_chat_message` - AI聊天消息表
- `heal_diary` - 疗愈日记表
- `user_health_info` - 用户健康信息表
- `user_info` - 用户基本信息表

### 2. 启动项目
```bash
# 进入项目目录
cd healthsystem-backend6/healthsystem-backend

# 使用Maven启动
mvn spring-boot:run

# 或者使用IDE直接运行 BackendApplication.java
```

### 3. 访问系统
- 后台管理系统：http://localhost:8080
- Swagger API文档：http://localhost:8080/swagger-ui.html
- Knife4j API文档：http://localhost:8080/doc.html

## 功能测试

### 一、AI聊天分析模块测试

#### 1. 访问页面
- 登录后台管理系统
- 点击左侧菜单 "AI聊天分析"
- 页面应正常加载，显示统计卡片和数据表格

#### 2. 测试统计功能
- 检查页面顶部的4个统计卡片是否显示数据
  - 正常（绿色）
  - 轻度风险（黄色）
  - 中度风险（橙色）
  - 重度风险（红色）

#### 3. 测试列表功能
- 检查表格是否显示数据
- 验证列：会话ID、用户姓名、用户名、手机号、对话标题、消息数量、创建时间、最后更新
- 测试分页功能（切换页码、修改每页显示数量）
- 测试排序功能（点击列标题排序）

#### 4. 测试搜索功能
- 在搜索框输入用户名或真实姓名
- 点击"搜索"按钮
- 验证搜索结果是否正确

#### 5. 测试查看对话功能
- 点击某条记录的"查看对话"按钮
- 弹窗应显示完整的对话记录
- 验证用户消息和AI消息的区分（颜色、标识）
- 验证消息时间戳显示

#### API测试
```bash
# 获取会话列表
curl -X GET "http://localhost:8080/aiChatAnalysis/getSessionList?currentPage=1&pageSize=15" \
  -H "Authorization: YOUR_TOKEN"

# 获取会话消息
curl -X GET "http://localhost:8080/aiChatAnalysis/getSessionMessages?sessionId=2" \
  -H "Authorization: YOUR_TOKEN"

# 获取情感统计
curl -X GET "http://localhost:8080/aiChatAnalysis/getEmotionStatistics" \
  -H "Authorization: YOUR_TOKEN"
```

## 常见问题排查

### 1. 页面无法加载
- 检查后端服务是否启动
- 检查浏览器控制台是否有错误
- 检查网络请求是否成功（F12 -> Network）

### 2. 数据不显示
- 检查数据库中是否有数据
- 运行 `新增功能数据库检查.sql` 查看数据统计
- 检查API接口返回的数据格式

### 3. 统计数据为0
- 确认数据库中有相应的数据
- 检查SQL查询逻辑是否正确
- 查看后端日志是否有错误

### 4. 搜索功能不工作
- 检查搜索关键词是否正确
- 验证SQL的LIKE查询是否正确
- 检查是否有特殊字符导致SQL错误

### 5. 权限问题
- 确认已登录且token有效
- 检查Shiro配置是否正确
- 查看后端日志的权限验证信息
